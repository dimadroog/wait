# TASK_BLANK — каркас задачи

Скопируйте в `docs/tasks/TASK_<SHORT_NAME>.md` (только **открытые** задачи).  
Закрытые и legacy — в [`archive/`](archive/). Отдельные `ISSUE_*` **не заводим**: гипотезы и прогоны — секции внутри TASK.

**Не класть в промпт по умолчанию:** `docs/tasks/archive/` (только если задача снова актуальна).

## Жизненный цикл

| Состояние | Где лежит |
| --------- | --------- |
| open | `docs/tasks/TASK_….md` |
| done / legacy | `docs/tasks/archive/TASK_….md` |

Расследования ведутся **только в TASK**. Архив бывших ISSUE: [`archive/ISSUE_*.md`](archive/).

---

## Алгоритм: перенос в archive (перелинковка)

Делать при `Статус: done` (или закрытии кампании).

1. **`git mv`** `docs/tasks/TASK_FOO.md` → `docs/tasks/archive/TASK_FOO.md`  
   Имя файла **не менять** без нужды (проще искать и чинить ссылки).
2. В шапке архивной копии: `Статус: done`, дата закрытия, одна строка «итог».
3. **Поиск ссылок** по репо (как минимум `docs/`, `.cursor/`, `scripts/`, `src/`):
   - `TASK_FOO.md`
   - `docs/tasks/TASK_FOO.md`
   - старые якоря секций, если на них ссылались снаружи
4. **Обновить** каждое попадание на `docs/tasks/archive/TASK_FOO.md` (или относительный путь от места ссылки).  
   Из `docs/*.md` → `tasks/archive/TASK_FOO.md`.  
   Из `docs/tasks/archive/*` → `TASK_FOO.md` (сосед) или `../../…` для концепт-доков.
5. Убедиться, что **нет** stub-файла в `docs/tasks/` с тем же именем (open-папка только для open).
6. Не подключать archive в новые промпты без явной нужды.

**Кампания / толстый журнал** (как бывший BACKLOG): один файл в archive, например [`TASK_FIRST_CAMPAIGN.md`](archive/TASK_FIRST_CAMPAIGN.md); не дробить задним числом на мелкие TASK.

## Алгоритм: новая задача

1. Нужен ли постоянный артефакт на несколько окон? Если нет — чеклист в чате, файл не создавать.
2. Скопировать этот blank → `TASK_<SHORT_NAME>.md`.
3. Заполнить цель, файлы, чеклист (≤ ~15 пунктов; иначе разрезать на 2 TASK). В шапке указать **`Ветка:`** `task/<short-name>` — проработку вести только в ней.
4. **Новая задача инициализирует новую git-ветку** от актуального `main`: `git checkout -b task/<short-name>`. Файл задачи (`docs/tasks/TASK_….md`) помещается в эту ветку **первым коммитом** (без посторонних правок). Дальнейшая проработка — только в этой ветке.
5. В чат: `@docs/tasks/TASK_….md` + файлы из шпаргалки — **не** весь archive.
6. По ходу: `- [x]`; при DoD → алгоритм archive выше.
